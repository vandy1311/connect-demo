02 Panel Triage at Scale

One command. N submissions. Human judgment on what matters.

  Via Kiro (natural language)
"Audit every repo in submissions/ and summarize the red ones."

Installable Kiro Power. Activates on keywords: audit, verify, hackathon, validate.

  Via CLI (CI, scripts, non-Kiro)
for repo in submissions/*/; do
  bash repo-audit.sh "$repo" \
    --quiet --format json --auto-heal
done

Same engine. Works anywhere bash works. Exit codes for gating.

Actionable Report

Pass/fail table with error column, Failure Details per check, pattern-matched fix suggestions, collapsible log excerpts.

JSON for Dashboards

--format json emits a self-contained report per repo — drop into any panel dashboard or triage spreadsheet.

Exit Codes for Gating

0 clean, 1 failures remain, 2 usage error. CI-friendly. Gate submissions before human review.

~90s Per Repo

8 checks, per-repo venv isolation means parallel audits are safe. Scales to the whole submission pool.

Scales the panel's read — human time for judgment, not plumbing. Every submission lands with a pre-audited report: what works, what's broken, and what to do about it.

Auto-Heal: false — humans fixed it, the tool guided them. Diagnostic-grade, not just an auto-fixer.

Panel-Grade Hardening — Why Trust It at Scale

  • Eval injection fix: error capture used eval() → replaced with a temp-file pattern (set_error / get_error). Tool won't execute untrusted error strings from submissions.
  • JSON escaping: json_escape via json.dumps for every untrusted string written to the report. Safe to pipe to dashboards.
  • Idempotent healing: running --auto-heal twice no longer stacks side effects. Safe to rerun on every submission.
  • Per-repo venv isolation: hash-scoped .audit-venv-<hash> — audit the whole submission pool in parallel without interference.
  • Fix suggestions built in: every failed check pairs the error with a pattern-matched remediation hint. Panel sees what broke and what to do.
Proof of value + proof of readiness. We caught real issues in our own repo (the tool works), then closed the security + safety gaps so the panel can run it on every submission with confidence.
